# 大会GO！ UIデザイン基本方針 v1.1

> **本文書の目的**: Claude Codeで開発を行うスタッフが、UIに関する判断で迷わず、統一感のあるインターフェースを構築するための基本方針。  
> **対象**: 開発チーム全員（Claude Codeのプロジェクト指示としても使用）  
> **最終更新**: 2026年3月  
> **参照ワイヤーフレーム**: rakusyo-go-wireframe.html（トップページ改修案）

---

## 1. デザイン原則（迷ったときの判断基準）

以下の5原則は優先順位順。矛盾する場合は上位の原則を優先する。

### 原則1: モバイルファースト・現場ファースト
- すべてのUIはスマートフォン（幅375px）から設計を開始する
- PC表示はモバイルの拡張であり、モバイルでの体験を劣化させてPC版を作らない
- 炎天下・片手操作・手袋着用でも使えることを前提とする

### 原則2: 2タップで目的に到達する
- 初見のユーザーが説明なしに操作できるUIを目指す
- **目的の大会・試合結果に2タップ以内で到達させる**
- 「都道府県→競技→さらに絞り込む」のような多段階クリックは禁止
- 操作結果は必ず視覚的フィードバックを即座に返す（ローディング、成功、エラー）

### 原則3: 「動いているシステム」を演出する
- 大会数・試合数がゼロの状態を無機質に表示しない
- ライブ感・活気を視覚的に表現する（リアルタイム速報、ステータスバッジ）
- 「過疎サイト」に見える表示は徹底的に排除する

### 原則4: 一貫性を保つ
- 同じ意味のUIは、どの画面でも同じ見た目・同じ挙動にする
- 新しいパターンを作る前に、既存のパターンで対応できないか確認する
- コンポーネント名・クラス名・変数名の命名規則を統一する

### 原則5: 信頼感と親しみやすさを両立する
- デザインは「堅すぎず、軽すぎず」を基調とする
- 信頼感（協会・組織が使うシステム）と親しみやすさ（子供の保護者も使う）の両立
- トップページの品格を保ち、バナーのベタ貼りで安っぽくしない
- スポーツの躍動感をカラーとタイポグラフィで表現する

## 2. デザイントークン

### 2.1 カラーパレット

ワイヤーフレームの配色体系に基づく。Tailwind CSSのカスタムカラーとして定義し、ハードコードされたカラーコードは使用禁止。

```
【ベースカラー】ヘッダー・フッター・重要な背景に使うダーク系
base-800:  #2c3e50   ← ヘッダー、フッター背景
base-700:  #34495e   ← サイドバー背景
base-600:  #3d566e   ← ホバー時のダーク背景
base-900:  #1a252f   ← 最も濃い背景（フッター下部など）

【プライマリ】アクセントカラー（ボタン・リンク・インタラクティブ要素）
primary-50:   #e3f2fd   ← アイコン背景、ホバー背景（薄い）
primary-100:  #bbdefb   ← 選択状態の背景
primary-400:  #42a5f5   ← ライトなアクセント
primary-500:  #3498db   ← メインのアクセントカラー（ボタン、リンク、アクティブタブ）
primary-600:  #2980b9   ← ホバー時のボタン
primary-700:  #2471a3   ← アクティブ（押下）時のボタン

【グラデーション】ヒーローセクション・ライブダッシュボードの背景
hero-gradient:    linear-gradient(135deg, #667eea 0%, #764ba2 100%)  ← ヒーロー・CTA背景
live-gradient:    linear-gradient(135deg, #f093fb 0%, #f5576c 100%)  ← ライブセクション背景
※ グラデーションは上記2パターンのみ使用。むやみに増やさない。

【セマンティック（意味のある色）】※ 必ずこの用途にのみ使う
success:   #27ae60   ← 成功・完了・勝利・LIVEバッジ
warning:   #f39c12   ← 注意・進行中
error:     #e74c3c   ← エラー・失敗・敗北・試合終了バッジ
info:      #3498db   ← 情報・お知らせ・更新バッジ

【ニュートラル（グレースケール）】
gray-50:   #f8f9fa   ← セクション背景（交互配色用）
gray-100:  #f0f0f0   ← プレースホルダー画像背景、区切り
gray-200:  #e0e0e0   ← ボーダー、カード枠線、divider
gray-300:  #d1d5db   ← disabled状態のボーダー
gray-400:  #999999   ← プレースホルダーテキスト
gray-500:  #666666   ← 補足テキスト、キャプション、説明文
gray-600:  #4b5563   ← セカンダリテキスト
gray-800:  #333333   ← プライマリテキスト（本文）
gray-900:  #2c3e50   ← 見出しテキスト（base-800と統一）

【テキストカラー（白背景上）】
- 見出し: gray-900 (#2c3e50)
- 本文: gray-800 (#333333)
- 補足: gray-500 (#666666)
- リンク: primary-500 (#3498db)

【テキストカラー（ダーク背景上）】
- メインテキスト: #ffffff
- サブテキスト: #ecf0f1
- 淡いテキスト: #95a5a6
- リンク: primary-500 (#3498db)
```

#### カラー使用ルール

| 用途 | 使うべき色 | 禁止 |
|------|-----------|------|
| CTAボタン（主要アクション） | primary-500 → hover: primary-600 | グレー系のCTA |
| 破壊的操作（削除、取消） | error（赤） | primary（青） |
| テキストリンク | primary-500 | 下線なしのリンク |
| ヘッダー/フッター背景 | base-800 | 白やグレーのヘッダー |
| ヒーロー背景 | hero-gradient | 単色の背景 |
| ライブセクション背景 | live-gradient | 他のグラデーション |
| セクション背景（交互） | white / gray-50 を交互に | 全セクション同色 |
| カード枠線 | gray-200、hover時にprimary-500 | 枠線なしのカード |
| 試合結果：勝利 | success | - |
| 試合結果：敗北 | error | - |
| 試合結果：引分 | gray-500 | - |
| LIVEバッジ | success背景 + 白テキスト | - |
| 試合終了バッジ | error背景 + 白テキスト | - |
| 更新バッジ | primary-500背景 + 白テキスト | - |

> **重要**: Tailwindのconfigで `colors` を一元定義し、コード内で直接hex値を書かない。  
> bg-background / text-foreground 等のセマンティッククラスを使い、ダークモード互換性を維持する。

### 2.2 タイポグラフィ

```
【フォントファミリー】
表示用（見出し・ロゴ周辺）: "M PLUS Rounded 1c", "Zen Maru Gothic", sans-serif
本文用: "Noto Sans JP", "Hiragino Sans", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif
数値・スコア表示: "Inter", "Noto Sans JP", sans-serif（等幅感のある数字が重要）

【フォントサイズスケール】（Tailwindクラス名で統一）
text-xs:   0.75rem (12px)  ← バッジ内テキスト、ティッカーのバッジ
text-sm:   0.875rem (14px) ← 補足テキスト、フッターリンク、タグライン
text-base: 1rem (16px)     ← 本文、検索入力、フィルターラベル（これが基準）
text-lg:   1.125rem (18px) ← ヒーローサブタイトル、導入事例テキスト
text-xl:   1.25rem (20px)  ← 機能紹介タイトル、導入事例の効果数値
text-2xl:  1.5rem (24px)   ← ロゴテキスト、ページ見出し（モバイル）
text-3xl:  1.875rem (30px) ← セクション見出し
text-4xl:  2.25rem (36px)  ← ヒーロータイトル（PC）
text-5xl:  3rem (48px)     ← ライブ統計の数字

【行間】
本文: leading-relaxed (1.625) ← 特に説明文、機能紹介の description
見出し: leading-tight (1.25)
UIラベル: leading-normal (1.5)
数値表示: leading-none (1) ← ライブ統計など大きな数字

【太さ】
normal (400): 本文、説明文
medium (500): UIラベル、ナビゲーション、タブ
semibold (600): 小見出し、フィルターラベル
bold (700): ページ見出し、セクションタイトル、スコア、統計数字、ロゴ
```

#### タイポグラフィ使用ルール
- 1画面内で使用するフォントサイズは最大4段階まで
- 太字（bold）は見出し・スコア・統計数字にのみ使い、本文中の強調には `semibold` を使う
- 全角英数字は使用しない（半角に統一）
- スコア表示（得点など）は常に `font-mono` または `tabular-nums` を適用する

### 2.3 スペーシング

```
【基本単位】4px（Tailwindのデフォルト: 1 = 0.25rem = 4px）

【セクションの内部余白】（ワイヤーフレーム基準）
ヒーロー・大セクション: py-16 px-10 (64px / 40px) → モバイル: py-10 px-4
カード内部: p-6 〜 p-8 (24px 〜 30px) → モバイル: p-4
検索コンテナ: p-8 (30px) → モバイル: p-4

【要素間の余白】
セクションタイトル → サブタイトル: mb-4 (16px)
サブタイトル → コンテンツ: mb-10 (40px)
カード内見出し → 説明文: mb-3 (12px)
フィルターグループ間: gap-5 (20px)

【グリッドのgap】
カードグリッド: gap-5 (20px) 〜 gap-8 (30px)
統計数値並び: gap-10 (40px)
フッターセクション間: gap-10 (40px)
```

#### スペーシングルール
- 余白は必ずTailwindのスペーシングスケールを使う（`p-4`, `mt-6`など）
- `margin-top: 13px` のような中途半端な値は禁止
- 要素間の余白は `gap` または `space-y/x` で管理し、個別マージンは最小限にする
- コンテンツエリアの最大幅: `max-w-screen-xl`（1280px）を標準、内部コンテンツは `max-w-4xl`（900px）

### 2.4 角丸・シャドウ・ボーダー

```
【角丸】
rounded (4px):     ボタン、入力フィールド、バッジ、ロゴ枠
rounded-lg (8px):  カード、モーダル、検索コンテナ、ドロップダウン
rounded-xl (12px): ティッカーバッジ（pill型）
rounded-full:      アバター、アイコンボタン（丸型）、アイコン背景

【シャドウ】
shadow-none:       デフォルトのカード（ボーダーで区切る）
shadow-sm:         ホバー前のカード（控えめ）
shadow-md:         検索コンテナ（0 4px 20px rgba(0,0,0,0.1)相当）
shadow-lg:         ホバー時のカード（0 6px 20px rgba(0,0,0,0.1)相当）

【ボーダー】
カード枠: border-2 border-gray-200 → hover: border-primary-500
セクション区切り: border-b-[3px] border-primary-500（ヘッダー下部のアクセントライン）
検索タブ下線: border-b-[3px] border-primary-500（アクティブタブ）
フッター内区切り: border-t border-white/10

※ ワイヤーフレームの特徴: カードは shadow ではなく border で輪郭を出す。
  hoverで border-color が primary に変わり、shadow-lg が付く。
```

## 3. コンポーネントルール

すべてshadcn/uiをベースとし、カスタムコンポーネントは必要最小限とする。

### 3.1 ボタン

```
【バリエーションと使い分け】

default（primary）: メインのアクション（保存、登録、送信、検索）
  → bg-primary-500 text-white rounded
  → 1画面に1つだけ（最も重要なアクション）

secondary: 補助的なアクション（キャンセル、戻る、フィルター適用）
  → bg-transparent text-white border-2 border-white（ダーク背景上）
  → bg-secondary text-secondary-foreground（ライト背景上）

destructive: 破壊的操作（削除、退会）
  → bg-error text-white
  → 必ず確認ダイアログを挟む

outline: 優先度の低いアクション（詳細を見る、もっと読む）
  → border border-input bg-background

ghost: ナビゲーション内・テーブル内のアクション
  → 背景なし、ホバーで薄い背景
```

#### ボタンサイズルール
```
sm:       テーブル内、コンパクトなUI（高さ32px）
default:  標準的なフォーム（高さ40px）
lg:       主要CTA、モバイルでの主操作、検索ボタン（高さ48px、最小タップ領域44px確保）
```

#### ボタン共通ルール
- ボタンラベルは動詞始まり（「保存する」「削除する」「大会を作成」）
- ヘッダーのCTA: 「大会を探す・結果を見る」（一般向け）と「主催者ログイン / 新規登録」（管理者向け）を明確に分離
- アイコン付きボタンは、アイコン（左）+ テキスト の配置を標準（例: 🔍 検索）
- ローディング中はボタンを `disabled` にし、スピナーを表示する
- モバイルでのタップ領域は最低44x44pxを確保する

### 3.2 フォーム・検索

```
【検索入力（トップページ・ヒーロー内）】
- 白背景の大きなコンテナ（rounded-lg shadow-md）内に配置
- タブ切り替え: 「フリーワード検索」「絞り込み検索」
- アクティブタブ: text-primary-500 + border-b-[3px] border-primary-500
- 検索入力: 大きめ（py-4 text-base）、border-2 border-gray-200
- プレースホルダー: 具体的な例を示す（「チーム名・大会名で検索（例：〇〇FC、パンダパンダ杯）」）
- フィルター: grid-cols-2（競技アイコン / 地域セレクト）

【競技アイコンセレクタ】
- 60x60px の正方形、border-2 border-gray-200 rounded-lg
- hover: border-primary-500 bg-primary-50
- 選択中: border-primary-500 bg-primary-50
- 内部にスポーツ絵文字（⚽🏀🏐⚾🏆）

【一般フォーム入力フィールド】
- ラベルは入力欄の上に配置（左揃え、font-semibold）
- 必須項目は ラベル + 赤い「*」で表示
- プレースホルダーはヒント（例: "チーム名を入力"）として使い、ラベルの代替にしない
- エラーメッセージは入力欄の直下に赤字で表示する
- 入力中のフィールドは border-primary-500 で囲む
- 入力フィールド: padding 12〜15px, border-2 border-gray-200, rounded

【バリデーション】
- Zod スキーマで一元定義し、フロント・サーバー両方で検証する
- リアルタイムバリデーションは blur 時（フィールドを離れた時）に発火
- 送信時に全フィールドを再検証する
- エラーメッセージは具体的に（「入力してください」ではなく「チーム名は必須です」）

【フォームレイアウト】
- モバイル: 1カラム（full width）
- PC: 最大2カラム（関連するフィールドのみ横並び、例: 姓+名）
- 送信ボタンはフォームの末尾、右揃え（モバイルはfull width）
```

### 3.3 カード

```
【標準カード】
- bg-white border-2 border-gray-200 rounded-lg
- padding: p-6〜p-8
- ホバー時: translateY(-4px) + shadow-lg + border-primary-500
- transition: all 0.3s
- カーソル: cursor-pointer（リンクカードのみ）

【主催者カード】（トップページ「注目の主催者」エリア）
- 中央揃えレイアウト
- ロゴ画像: 120x80px, bg-gray-100, rounded, 中央配置
- 主催者名: font-bold, text-gray-900
- 補足情報: text-sm text-gray-500（「開催中: 3大会 / 募集中: 2大会」）

【大会カード】（大会一覧で使用）
- 上部にステータスバッジ（募集中/開催中/完了）
- 大会名は text-lg font-semibold で最も目立つ
- 主催者・日程・会場は text-sm text-gray-500
- 部門数はバッジで表示

【試合結果カード / ティッカーアイテム】
- 背景: rgba(255,255,255,0.9)（ダーク/グラデーション背景上）または bg-white
- padding: p-3
- rounded
- flex justify-between items-center
- 大会名+対戦カード（左）、ステータスバッジ（右）
- スコアは font-bold で強調

【機能紹介カード】
- border-2 border-gray-200 rounded-lg, text-center
- アイコン: 80x80px rounded-full bg-primary-50, 中央配置, 絵文字 text-4xl
- タイトル: text-xl font-bold text-gray-900
- 説明文: text-gray-500 leading-relaxed

【導入事例カード】
- 上部: ビジュアルエリア（高さ200px, hero-gradient背景, 大会名を白テキスト中央配置）
- 下部: padding p-6
  - 主催者名: text-lg font-bold
  - 効果数値: text-xl font-bold text-error（「運営スタッフが50%削減！」）
  - 説明文: text-gray-500 leading-relaxed
```

### 3.4 バッジ・ステータス表示

```
【ステータスバッジ】
- 共通: px-3 py-1 rounded-xl text-xs font-bold text-white
- LIVE / 開催中: bg-success
- 試合終了 / 完了: bg-error
- 更新: bg-primary-500
- 募集中: bg-warning
- 開催前: bg-gray-400

【数値が0のときの処理】★重要
- 「開催中の大会数: 0」「募集中: 0」のような表示は禁止
- ゼロの場合は該当セクション自体を非表示にするか、
  「次の大会をお楽しみに！」のようなポジティブな代替テキストを表示
- 統計カードにゼロが並ぶ「過疎サイト」の印象を絶対に避ける
```

### 3.5 テーブル

```
【基本ルール】
- ヘッダー行: bg-gray-50 font-medium text-gray-500 text-sm
- データ行: hover:bg-gray-50 で行全体がハイライト
- ボーダー: 行間に border-b border-gray-200（縦罫線は使わない）
- セル内のテキスト: text-sm
- 数値は右揃え、テキストは左揃え

【モバイル対応】
- 横スクロール禁止（モバイルでテーブルが横にはみ出してはいけない）
- モバイルではテーブルをカード形式に変換するか、重要列のみ表示する
- 順位表（リーグ）は横スクロール許可（例外）だが、チーム名列は固定する
```

### 3.6 モーダル・ダイアログ

```
【使う場面】
- 確認が必要な破壊的操作（削除、取消）
- 簡単な入力フォーム（項目3つ以下）
- 重要な通知（エラー、完了）

【使わない場面】
- 複雑なフォーム → 別ページに遷移する
- 情報の表示のみ → トーストまたはページ内で表示
- ネストされたモーダル（モーダルの中からモーダルを開かない）

【デザインルール】
- オーバーレイ: bg-black/50（半透明黒）
- モーダル本体: bg-white rounded-lg shadow-lg max-w-md
- 閉じる手段: ×ボタン + オーバーレイクリック + Escキー（すべて対応）
- アクションボタン: 右下配置、キャンセル（左）+ 確定（右）
```

### 3.7 トースト・通知

```
【用途別】
success: 操作完了（保存しました、登録しました）→ 3秒で自動消去
error:   操作失敗（保存に失敗しました）→ 手動で閉じるまで残る
warning: 注意喚起（締切が近づいています）→ 5秒で自動消去
info:    お知らせ（新しい大会が追加されました）→ 5秒で自動消去

【位置】
モバイル: 画面上部中央
PC: 画面右上

【ルール】
- 同時表示は最大3つまで（古いものから消える）

```

## 5. 画面タイプ別ガイドライン

### 5.1 トップページ（ランディング）

- **最重要**: 検索機能をファーストビューに配置。保護者は「〇〇FC」で検索する
- **3つの導線を明確に**: ヘッダーで「大会を探す」と「主催者ログイン」を分離
- **主催者ポータル**: クリック1回で主催者専用ページ（/organizer/[id]）に遷移
- **ライブ感の演出**: ライブダッシュボードで「動いているシステム」を表現
- **0表示禁止**: 大会数がゼロの項目はセクションごと非表示
- **品格の維持**: オフィシャルパートナーはフッターに品良く配置、バナーのベタ貼り禁止

### 5.2 一般公開ページ（大会一覧、大会詳細、試合結果）

- **目的**: 情報をすばやく見つけて理解する
- **トーン**: 明るく、スポーティ、見やすい
- **背景**: セクション交互配色（white / gray-50）
- **カードベース**: 情報はカードで区切り、スキャンしやすく
- **CTA**: 「チーム登録」「大会に申し込む」を最も目立たせる

### 5.3 管理画面（大会管理者向け）

- **目的**: 効率的にデータを入力・管理する
- **トーン**: 機能的、プロフェッショナル、落ち着いた
- **背景**: white（サイドバーはgray-50）
- **テーブル多用**: データ管理はテーブル形式を基本とする
- **バルク操作**: 複数選択+一括操作をサポートする

### 5.4 試合進行画面（QRコード経由の審判入力）

- **目的**: 最速でスコアを入力する
- **トーン**: 高コントラスト、大きなUI、ミニマル
- **特別ルール**:
  - フォントサイズを通常の1.5倍にする
  - ボタンは最低56x56pxのタップ領域
  - 不要な要素は徹底的に排除する
  - 入力ミスを防ぐ確認ステップを必ず挟む
  - 屋外での視認性のため、高コントラスト配色

## 6. アニメーション・トランジション

```
【基本方針】
- アニメーションは「意味のある変化」にのみ使う
- 装飾的なアニメーションは原則使わない
- 所要時間は 150ms 〜 300ms の範囲内

【使うべきアニメーション】
- カードホバー: translateY(-4px) + shadow-lg, transition all 0.3s ← ワイヤーフレーム準拠
- ページ遷移: フェードイン (opacity 0→1, 150ms)
- モーダル表示: フェードイン + スケール (scale 0.95→1, 200ms)
- トースト表示: スライドイン (translateY, 200ms)
- アコーディオン開閉: 高さのトランジション (200ms)
- 検索タブ切り替え: フェード (150ms)
- スポーツアイコン hover: border-color + background, transition all 0.2s

【使わないアニメーション】
- バウンス、揺れ、回転などの過度な演出
- 3秒以上かかるアニメーション
- 自動再生のスライドショー（ユーザー操作で切り替え）
- スクロールに連動するパララックス
```

## 7. アイコン

スポーツ競技アイコン: 絵文字（⚽🏀🏐⚾🏆）を使用

## 8. ダークモード対応方針

```
【現時点の方針】
- 既存のダークモード（テーマ切り替え機能）は維持する
- ただし、ダークモードの改善・最適化は優先度を下げる
  → コア機能の開発・ライトモードのUI統一を優先する
- 新規画面・コンポーネント作成時のルール:
  - 色はすべてCSS変数（Tailwindのテーマ）で管理する
  - ハードコードされた色（bg-white, text-blackなど）は使わず、
    セマンティックなクラス（bg-background, text-foreground）を使う
  - shadcn/uiのテーマ機能をそのまま活用する
  - これにより、ダークモードが「壊れない」状態を最低限保つ
- ダークモード専用のデザイン調整（色味の微調整、コントラスト最適化など）は
  コア機能が安定した段階で改めて取り組む
```

## 9. アクセシビリティ最低基準

```
【必須対応】
- カラーコントラスト比: WCAG AA準拠（テキスト 4.5:1以上、大文字 3:1以上）
- すべてのインタラクティブ要素にフォーカスインジケーターを表示
- 画像には alt テキストを設定（装飾画像は alt="" で空にする）
- フォーム要素には label を紐付ける
- キーボードだけで主要操作が完了できる
- スクリーンリーダーで情報が正しく読み上げられる

【推奨対応】
- aria-live でスコア更新を通知
- 色だけでなくアイコン・テキストでも状態を伝える（色覚多様性対応）
```

## 11. 絶対に避けるべきこと（ワイヤーフレーム設計メモより）

```
❌ 階層構造の罠
  「都道府県を選ぶ → 競技を選ぶ → さらに絞り込む」のような多段階クリックは禁止。
  2タップ以内でリストに到達させる。

❌ 管理者目線の数字
  「募集中の大会数：0」のような表示は削除。ゼロが並ぶと「過疎サイト」に見える。
  データがない状態はポジティブな表現に変換するか、セクション自体を非表示にする。

❌ バナーベタ貼り
  トップページに広告バナーを無秩序に並べると安っぽくなる。
  オフィシャルパートナーはフッターに品良く配置。
  大会ページ内の「ローカルスポンサー」とは明確に区別する。

❌ 横スクロールするテーブル
  モバイルでテーブルが横にはみ出してはいけない（順位表を除く）。

❌ モーダルの入れ子
  モーダルの中からモーダルを開かない。

❌ ハードコードされたカラーコード
  bg-[#3498db] のようなハードコードは禁止。テーマトークンを使う。
```

## 12. 品質チェックリスト

新しい画面・コンポーネントを作成する際に確認する。

### 全画面共通
- [ ] モバイル（375px幅）で表示崩れがないか
- [ ] タブレット（768px幅）で適切なレイアウトになっているか
- [ ] タップ領域が44x44px以上確保されているか
- [ ] ローディング状態が表示されるか
- [ ] エラー状態が適切に表示されるか
- [ ] 空状態（データなし）にポジティブな表現があるか（ゼロ表示禁止）
- [ ] カラーはデザイントークンのみ使用しているか（ハードコード禁止）
- [ ] フォントサイズはTailwindのスケールのみ使用しているか
- [ ] セマンティックカラークラス（bg-background等）を使用しダークモードが壊れないか
- [ ] カードのホバーエフェクト（translateY + shadow + border色変更）が動作するか

### フォーム
- [ ] バリデーションエラーが入力欄の直下に表示されるか
- [ ] 必須項目に赤い「*」があるか
- [ ] 送信中はボタンがdisabledになり、スピナーが表示されるか
- [ ] 送信成功後にトーストまたはリダイレクトがあるか

### テーブル・一覧
- [ ] モバイルで横スクロールが発生していないか（例外: 順位表）
- [ ] 空状態の表示があるか
- [ ] ソート・フィルターが機能しているか（該当する場合）

## 13. Claude Codeへの指示テンプレート

上記の方針をClaude Codeのプロジェクト指示に組み込む際の要約版：

```
【UIデザイン指示（Claude Code用）】

■ 技術スタック
- Next.js 15 (App Router) + React 19 + TypeScript
- Tailwind CSS 4 + shadcn/ui
- アイコン: lucide-react（スポーツ競技は絵文字）
- フォーム: React Hook Form + Zod
- フォント: M PLUS Rounded 1c（見出し）, Noto Sans JP（本文）, Inter（数値）

■ 基本原則
1. モバイルファースト（375px幅から設計）
2. 2タップで目的に到達（多段階クリック禁止）
3. 「動いているシステム」を演出（ゼロ表示禁止、ライブ感演出）
4. 色・フォントサイズ・余白はすべてTailwindのテーマトークンを使用
5. ハードコードされたカラーコード禁止（bg-background/text-foreground等を使う）

■ カラー体系
- ベース（ヘッダー/フッター）: #2c3e50系ダークネイビー
- プライマリ（ボタン/リンク）: #3498db系ブルー
- ヒーロー背景: linear-gradient(135deg, #667eea, #764ba2)
- ライブ背景: linear-gradient(135deg, #f093fb, #f5576c)
- セクション背景: white と #f8f9fa の交互配色
- セマンティック: success=#27ae60, error=#e74c3c, warning=#f39c12

■ コンポーネント
- UIコンポーネントはshadcn/uiを使う
- カスタムCSSは原則禁止（Tailwindユーティリティで解決）
- カード: border-2 border-gray-200 → hover: translateY(-4px) shadow-lg border-primary
- ボタン: default(CTA), secondary(白枠線/ダーク背景用), destructive, outline, ghost
- 破壊的操作には必ず確認ダイアログ
- バッジ: LIVE=緑, 終了=赤, 更新=青, 募集中=黄

■ レイアウト
- コンテンツ幅: max-w-screen-xl mx-auto px-4（内部コンテンツ max-w-4xl）
- 管理画面はサイドバー付きレイアウト
- グリッド: repeat(auto-fit, minmax(250px, 1fr))
- ヘッダー: ダーク背景 + 下部にprimary色アクセントライン

■ 禁止事項
- ゼロが並ぶ統計表示（「募集中: 0」等）→ 非表示 or ポジティブ代替テキスト
- 多段階の階層ナビゲーション（2タップ以内）
- 横スクロールが発生するテーブル（順位表を除く）
- モーダルの中からモーダルを開く
- 3秒以上のアニメーション
- アイコンだけのボタン（aria-labelなし）
- bg-white, text-black等のハードコード色
- トップページへの広告バナーのベタ貼り
```
