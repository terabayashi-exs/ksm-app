# システム全体像ガイド

KSM-App（PK選手権大会運営システム）の業務フロー、ユーザーロール、画面構成の全体像です。
新しくプロジェクトに参加するメンバーは、まずこのドキュメントを読んでシステムの目的と流れを理解してください。

## システムの目的

PK選手権をはじめとするスポーツ大会の**運営業務を一元管理する**Webアプリケーションです。
大会の作成から、チーム募集、組合せ抽選、試合進行、結果公開、アーカイブまでの全フローをカバーします。

## ユーザーロール

本システムには4種類のユーザーがいます。

| ロール | 対象者 | 主な画面 | 認証 |
|--------|--------|----------|------|
| **管理者 (Admin)** | 大会主催者・運営責任者 | `/admin/*` | メール + パスワード |
| **オペレーター (Operator)** | 大会当日の運営スタッフ | `/admin/*`（権限制限あり） | メール + パスワード（招待制） |
| **チーム代表者 (Team)** | 参加チームの代表者 | `/my/*`, `/tournaments/*` | メール + パスワード |
| **審判 (Referee)** | 試合の審判 | `/referee/*` | QRコード認証（ログイン不要） |
| 一般閲覧者 | 試合結果を見る人 | `/public/*` | 認証不要 |

### 管理者 (Admin)
- システムの全機能にアクセス可能
- 大会グループ・部門の作成と管理
- フォーマット（試合テンプレート）の作成
- オペレーターの招待と権限設定
- チーム・選手の管理
- 試合結果の確定とアーカイブ
- `isSuperadmin` フラグを持つ管理者は、非公開フォーマットへのアクセスなど追加権限あり

### オペレーター (Operator)
- 管理者から招待されて参加
- **大会ごとに18種類の個別権限**を設定可能（例: 結果入力のみ、チーム登録のみ、など）
- 管理者画面（`/admin/*`）にアクセスできるが、許可された操作のみ実行可能
- 権限プリセット: 「事前準備」「当日運営」「全権限」など

### チーム代表者 (Team)
- メール認証で自己登録
- 複数チームを管理可能（`m_team_members` テーブル）
- 大会への参加申請・選手登録
- 辞退申請

### 審判 (Referee)
- ログイン不要。管理画面から発行されるQRコードを読み取ってアクセス
- 試合のスコア入力のみ可能

## 大会運営の業務フロー

大会は以下の流れで運営されます。各ステップは自動的にステータスが遷移します。

```
┌─────────────────────────────────────────────────────────┐
│                    大会運営フロー                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 準備フェーズ                                         │
│  ┌───────────┐    ┌───────────┐    ┌───────────┐       │
│  │ 大会グループ │───▶│ 部門作成   │───▶│ ルール設定  │       │
│  │ 作成       │    │(フォーマット│    │(勝ち点,    │       │
│  │           │    │ 選択)     │    │ PK戦等)   │       │
│  └───────────┘    └───────────┘    └───────────┘       │
│        │                                                │
│        ▼                                                │
│  2. 募集フェーズ               ステータス: recruiting     │
│  ┌───────────┐    ┌───────────┐                        │
│  │ チーム募集  │───▶│ 選手登録   │                        │
│  │ 開始       │    │(CSV一括も可)│                        │
│  └───────────┘    └───────────┘                        │
│        │                                                │
│        ▼                                                │
│  3. 組合せフェーズ             ステータス: before_event   │
│  ┌───────────┐    ┌───────────┐    ┌───────────┐       │
│  │ 組合せ抽選  │───▶│ スケジュール│───▶│ QRコード   │       │
│  │ (ブロック   │    │ 確認      │    │ 発行       │       │
│  │  振り分け)  │    │           │    │           │       │
│  └───────────┘    └───────────┘    └───────────┘       │
│        │                                                │
│        ▼                                                │
│  4. 開催フェーズ               ステータス: ongoing        │
│  ┌───────────┐    ┌───────────┐    ┌───────────┐       │
│  │ 試合進行   │───▶│ スコア入力  │───▶│ 結果確定   │       │
│  │ (審判QR)   │    │ (リアルタイム│    │ (順位自動   │       │
│  │           │    │  更新)     │    │  計算)     │       │
│  └───────────┘    └───────────┘    └───────────┘       │
│        │                  │                             │
│        │                  ▼                             │
│        │          ┌───────────┐                        │
│        │          │ 予選→決勝  │                        │
│        │          │ 自動進出   │                        │
│        │          └───────────┘                        │
│        ▼                                                │
│  5. 完了フェーズ               ステータス: completed      │
│  ┌───────────┐    ┌───────────┐                        │
│  │ 結果公開   │───▶│ アーカイブ  │                        │
│  │ (公開画面)  │    │ (HTML保存) │                        │
│  └───────────┘    └───────────┘                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### ステータス遷移（自動判定）

ステータスは日付ベースで自動計算されます（`lib/tournament-status.ts`）。

| ステータス | 条件 | 画面表示 |
|-----------|------|---------|
| `planning` | 募集開始日前 | 準備中 |
| `recruiting` | 募集開始日 〜 募集終了日 | 募集中 |
| `before_event` | 募集終了日 〜 開催日前日 | 開催前 |
| `ongoing` | 開催日当日 〜 最終開催日 | 開催中 |
| `completed` | 最終開催日後、または全試合確定済み | 終了 |

## 画面構成マップ

### 管理者・オペレーター画面 (`/admin/*`)

```
/admin/
├── (ダッシュボード)              ← 大会一覧・通知
├── tournament-groups/           ← 大会グループ管理
│   ├── create/                  ← 新規大会グループ作成
│   └── [id]/                    ← 大会グループ詳細・編集
├── tournaments/                 ← 部門管理
│   ├── create-new/              ← 新規部門作成
│   └── [id]/                    ← 部門ごとの管理画面
│       ├── teams/               ←   チーム管理
│       ├── draw/                ←   組合せ抽選
│       ├── matches/             ←   試合管理（リアルタイム）
│       ├── results/             ←   結果確認
│       └── ...                  ←   ルール/ファイル/QR等
├── tournament-formats/          ← フォーマット管理
├── venues/                      ← 会場管理
├── teams/                       ← チームマスター管理
├── sport-types/                 ← 競技種別管理
├── operators/                   ← オペレーター管理
├── announcements/               ← お知らせ管理
└── subscription/                ← サブスクリプション管理
```

### チーム代表者画面 (`/my/*`)

```
/my/
├── (ダッシュボード)              ← 参加大会一覧
├── teams/                       ← 管理チーム一覧・選手管理
└── settings/                    ← アカウント設定
```

### 公開画面 (`/public/*`, `/tournaments/*`)

```
/public/tournaments/
├── groups/[id]/                 ← 大会グループページ
└── [id]/                        ← 部門詳細
    ├── (トップ)                  ← スケジュール・速報
    ├── bracket/                 ← トーナメント表
    └── archived/                ← アーカイブ表示

/tournaments/[id]/
├── join/                        ← 大会参加登録
└── withdrawal/                  ← 辞退申請
```

## データ構造の概要

### 「大会」と「部門」の関係

本システムでは、1つの大会イベントの中に複数の部門（カテゴリー）を持つ構造です。

```
大会グループ (t_tournament_groups)     ← 例: 「第5回PK選手権」
├── 部門A (t_tournaments)              ← 例: 「一般の部」
│   ├── 予選ブロックA (t_match_blocks)
│   ├── 予選ブロックB (t_match_blocks)
│   └── 決勝トーナメント (t_match_blocks)
└── 部門B (t_tournaments)              ← 例: 「ジュニアの部」
    └── ...
```

> **注意**: コード上では `Tournament` = 部門、`TournamentGroup` = 大会です。
> 詳しくは[用語集](./glossary.md)を参照してください。

### 試合データの2段階管理

| テーブル | 役割 | タイミング |
|---------|------|-----------|
| `t_matches_live` | 進行中の試合データ | スコア入力中 |
| `t_matches_final` | 確定済みの試合データ | 管理者が結果を確定した後 |

スコアは **JSON配列形式** で保存されます（例: `[3, 2, 1]` = 前半3点、後半2点、延長1点）。
詳細は[スコアパーサー仕様](../features/score-parser.md)を参照してください。

## 主要な技術的特徴

| 特徴 | 詳細 |
|------|------|
| **リアルタイム更新** | Server-Sent Events (SSE) で試合状況をリアルタイム表示 |
| **自動進出** | 予選結果に基づき、決勝トーナメントに自動でチームを配置 |
| **複数チーム参加** | 同一マスターチームが複数エントリー可能（`tournament_team_id` で識別） |
| **JST統一** | 全タイムスタンプは `datetime('now', '+9 hours')` でJST |
| **アーカイブ** | 完了した大会をHTML形式で完全保存（Vercel Blob Storage） |
